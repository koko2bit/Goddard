# State Module: WorkbenchTabsState
- **Responsibility:** Manage the lifetime of closable detail tabs, including open-or-focus reuse, active ordering, drag reorder, least-recently-used eviction, cached restoration, and restart persistence.
- **Data Shape:** One stable primary tab descriptor; one map of open detail tabs keyed by tab id with kind, icon, title, payload, and dirty status; one active tab id; one ordered array for strip rendering; one recency stack for LRU decisions; one cache of recently closed tab snapshots kept in memory; one persisted workspace snapshot. The detail-tab cap should be a policy field so the requested 10-tab target can be reconciled with the current `spec/app.md` 20-tab limit without changing component contracts.
- **Mutations/Actions:** `openOrFocusTab`; `activateTab`; `closeTab`; `closeLeastRecentlyUsedTab`; `reorderTabs`; `updateTabTitle`; `updateTabDirtyState`; `restoreCachedTab`; `hydrateTabsFromStore`; `persistTabs`.
- **Scope & Hoisting:** Hoisted into a global provider because tabs are shared across the entire shell and must survive view switches and app restarts.
- **Side Effects:** Persists tab snapshots to Tauri-backed storage; coordinates with feature state modules to keep hidden tabs alive in memory; enforces the detail-tab cap when opening new tabs; may translate domain actions such as “open PR” into a stable tab id and payload.
