# Workspace Agent Notes

- `AGENTS.md` files are for agent bootstrap and document-routing rules.
- `CONTRIBUTING.md` files are for project-specific contribution policy.
- Read the root and nearest local `AGENTS.md` and `CONTRIBUTING.md` before substantial work in a subtree.
- More local docs may add or narrow rules for their subtree.
- You may update any `AGENTS.md` in this workspace when a human gives durable feedback that future agents should follow.
- Keep each `AGENTS.md` short, generic, and scoped to its directory tree.
- Put package boundaries and integration surface in the nearest `README.md`.
- Put domain terminology in the nearest `glossary.md`.
- When printing relative paths for this workspace, use repository-relative paths.
- Do not use `AGENTS.md` as a spec, plan, backlog, or changelog.
- When guidance outgrows an `AGENTS.md`, move it into a better-scoped document and leave a short pointer.
