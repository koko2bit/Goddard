`goddard declare-initiative --title "<outcome-oriented-title>"`
   - Use this AFTER you have investigated the codebase and decided on a plan, but BEFORE you begin executing the actual changes.
   - Initiatives must be outcome-oriented goals, not low-level actions. Aim for one initiative per task/turn.
   - BAD: "Searching the codebase", "Reading files", "Writing code"
   - GOOD: "Fix null pointer in auth middleware", "Add pagination to user list"
